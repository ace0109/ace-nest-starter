# 贡献指南

## 概述

欢迎贡献代码！请遵循以下指南。

## 开发流程

1. Fork 项目
2. 创建特性分支
3. 提交更改
4. 推送分支
5. 创建 Pull Request

## 提交规范

使用 Conventional Commits:

- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

## 代码审查

- 代码质量
- 测试覆盖
- 文档更新
- 性能影响

## 贡献类型

- 报告 Bug
- 提出新功能
- 改进文档
- 提交代码
